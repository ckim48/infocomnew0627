# Figure source data (CSV)

Generated by `python3 -m sim.export_fig_csv`. One CSV per figure, exactly
the values that appear in the plots.

## seoul_map_realacc.csv → fig_seoul_map_realacc_loo(_compact)
One row per vehicle (180) from the 400-round extended replay
(`metrics_v2x_real_kitti_map400.npz`, seed 2026).
- `lon`, `lat` — WGS84 position at the map snapshot
- `x_web_mercator`, `y_web_mercator` — EPSG:3857 (for basemap tiles)
- `heading_deg` — instantaneous heading (0° = east, CCW)
- `acc_*` — per-vehicle final test accuracy under each scheme.
  Figure colormap: RdYlGn, vmin=0.05, vmax=0.95.

## seoul_deadline.csv → fig_seoul_deadline
`dataset` (kitti / nuscenes) x `scheme` (6) x `deadline_rounds`
(1,2,3,5,10,20): windowed P(useful encoder reaches a high-demand vehicle
within d rounds); mean/std over 3 seeds. Figure: x log-scale, y in [0, 0.8].

## seoul_calib.csv → fig_seoul_calib
Per dataset, FACE's out-of-sample gain prediction grouped into predicted-
gain deciles (warmup 30 rounds excluded): mean predicted vs mean realized
normalized validation gain, SEM of the realized mean. RAW units — the
figure multiplies both axes by 1e3.

## face_abl_2panel.csv → fig_face_abl_2panel(_sep)
Per ablation variant: final accuracy (all / high-demand vehicles, %,
mean±std over seeds) and communication volume split into useful vs
redundant deliveries (GB, sum over 250 rounds). `variant_label` is the
LaTeX-formatted x-tick label used in the figure.

## seoul_map_utility.csv → fig_seoul_map (abstract backend)
One row per vehicle (180): same positions/headings as
seoul_map_realacc.csv, but `utility_*` columns hold the ACHIEVED
STATISTICAL UTILITY q_eff of the abstract Seoul backend (best acquired
encoder strength; local training contributes nothing in this model, so
the map isolates pure propagation). Figure colormap: RdYlGn,
vmin=0.2, vmax=1.0.
